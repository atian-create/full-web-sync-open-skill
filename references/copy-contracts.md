# Copy Contracts

Use these contracts after project facts and GitHub state are verified.

## X Platform Text

Write copy that can be pasted directly into X.

Default shape:

```text
I open-sourced [project name].

It helps [target user] do [job] by [workflow].

Useful when:
- [use case 1]
- [use case 2]
- [use case 3]

GitHub:
[repo URL]
```

Rules:

- Keep it concise.
- Lead with the project fact.
- Use concrete use cases.
- Avoid fake metrics or hype.
- Include the GitHub URL when available.
- For Chinese audiences, Chinese copy is acceptable and often better.

## Knowledge Planet Copy

Write a fuller community post that explains what the project is and how to use it.

Default shape:

```text
【Open Source】[project name]

GitHub repo 已创建并公开：
[repo name]

这个 Skill / 项目是用来帮你 [main job].

适合这些场景：
1. [scenario]
2. [scenario]
3. [scenario]

安装以后，你可以这样问：
[example prompt]

它会输出：
- [deliverable]
- [deliverable]
- [deliverable]

GitHub:
[repo URL]
```

Rules:

- Make it copy-paste ready.
- Explain who should use it.
- Explain what the user gives it and what it returns.
- Keep project facts accurate.
- Do not over-promise outcomes.

## Optional Build In Public Copy

Use when the user wants public building notes.

Include:

- why this was made
- what changed from the internal workflow
- what was intentionally excluded for privacy
- what the first public version can do
- what may be improved next
